## 项目概览

`clawTest` 是一个基于 Python 的桌面小工具集，围绕 Word 文档处理场景，提供以下三大功能：

- **Word 校对工具**：读取 Word 文档内容，并基于“错词 → 正词”知识库进行批量替换校对。
- **知识库管理器**：维护专业术语和“错词 → 正词”映射的知识库，支持手工导入、候选错词生成与审核。
- **精简 Word 文档**：按“说话人(时间戳): 内容”格式，合并同一说话人连续多段话，便于整理对话记录（如心理催眠记录）。

推荐通过根目录下的 `.bat` 启动脚本在 Windows 上使用，无需手工在命令行中执行 Python。

## 目录结构总览

以下是项目的主要目录与文件（仅列出与功能相关的关键内容）：

- `run.py`  
  - 控制台启动入口，提供菜单：
    - `1`：启动 Word 校对工具（`src/word_reader.py`）
    - `2`：启动 知识库管理器（`src/errorLibrary/readLibrary.py`）

- `src/`
  - `word_reader.py`：Word 校对工具与精简工具的 GUI 主程序（Tkinter）。
  - `errorLibrary/`：错词知识库与相关工具。
    - `Word_Library.txt`：主“错词 → 正词”词库（文本源文件）。
    - `Word_Library.md`：从 `Word_Library.txt` 同步生成的 Markdown 版词库文档（适合阅读与查看）。
    - `Proper_Word_Library.txt`：专业术语列表（每行一个术语）。
    - `Pending_Review.txt`：待审核候选词条（目前为空，预留给后续使用）。
    - `readLibrary.py`：知识库管理器 GUI 主程序（维护术语与错词映射）。
    - `update_library.py`：将 `Word_Library.txt` 中的新词条同步到 `Word_Library.md`。
    - `素材文件/`：知识库相关的素材示例。
      - `library.txt`：原始词库样例，用于生成/维护 `Word_Library` 的基础数据。
      - `read_docx.py`：与 docx 读取相关的辅助脚本示例。
      - 其它示例文本（如 `111.txt`、`示例词库.txt`）。

- `docs/`
  - `用户操作手册.md`：面向最终用户的操作指南（安装、启动方式、各功能面板说明、常见问题）。
  - `项目结构与知识库说明.md`（本文件）：面向开发者/维护者的结构与知识库设计说明。

- 批处理脚本（Windows 启动/安装相关）
  - `install.bat`：安装 Python 依赖（如 `python-docx`）。
  - `run.bat`：提供菜单式启动入口（内部会调用 `run.py`）。
  - `启动Word校对.bat`：直接启动 Word 校对工具。
  - `启动知识库管理器.bat`：直接启动知识库管理器。
  - `config.bat`：可配置固定的 `PYTHON_PATH`。
  - `find_python.bat`：自动搜索本机 Python 安装路径。

- 文档与说明
  - `README.txt`：简要的“快速开始”和启动方式说明。

## 核心模块说明

### 1. 启动入口：`run.py`

- 以命令行菜单形式统一入口：
  - 选择 `1` 时：调用 `python src/word_reader.py`。
  - 选择 `2` 时：调用 `python src/errorLibrary/readLibrary.py`。
- 主要负责：
  - 给非技术用户一个简单的命令行菜单。
  - 统一启动路径，避免直接双击 `.py` 文件造成闪退。

### 2. Word 校对与精简：`src/word_reader.py`

- 使用 `tkinter` 构建桌面 GUI，分为：
  - **顶部按钮区**：
    - “📂 选择要被校准的文件”：选择需校对的 Word 文档（`.docx` / `.doc`）。
    - “🗑️ 清空内容”：清空当前内容。
  - **左侧内容区**：
    - 显示当前选中文档的文本内容。
  - **右侧按钮区**：
    - “🔧 执行校准”：根据知识库批量替换错词，生成新的 `xxx_校准.docx`。
    - “📚 知识库”：启动 `readLibrary.py` 打开知识库管理器。
    - “📂 选择被精简的Word” + “✂️ 执行精简”：按规则合并同一人连续说话的多段话，生成 `xxx_精简.docx`。
  - **底部日志区**：
    - 实时输出处理日志、替换详情、错误信息等。

- **知识库关联**：
  - 在初始化时默认将 `self.library_path` 设置为：  
    `src/errorLibrary/Word_Library.md`。
  - 校对时会：
    - 读取 `Word_Library.md`。
    - 使用正则解析其中的 `错误词=正确词` 条目，构建字典。
    - 对 Word 文档的段落与表格逐条执行字符串替换，并统计替换次数。

### 3. 知识库管理器：`src/errorLibrary/readLibrary.py`

`ReadLibraryApp` 是知识库管理 GUI 的核心类，主要职责分三部分：

1. **专业术语管理（术语 Tab）**
   - 数据源：`Proper_Word_Library.txt`。
   - 支持：
     - 批量添加术语（按空格/逗号/顿号/换行等分隔）。
     - 对已存在术语去重。
     - 编辑、删除术语。
     - 以“添加顺序”或“字母顺序”排序查看。

2. **候选错词生成与审核（预览 / 待审核 Tab）**
   - 基于 `CHAR_MAPPING` 映射：  
     为专业术语的某些关键字生成可能的听写/口音错误变体（候选错词）。
   - 工作流：
     - 通过 “🎯 全部生成候选错词” 或 “🎯 选中生成错词” 生成候选列表。
     - 在“待审核”列表中，通过点击切换每一条的“☐ / ☑”状态。
     - 使用“全选 / 取消全选 / 删除选中”等操作辅助筛选。
     - 通过“📥 批量导入词库”将选中的候选错词批量写入 `Word_Library.txt`。

3. **词库导入与浏览（导入预览 / 词库 / 文档 Tab）**
   - “📥 手工导入词库”：
     - 支持从 `.txt` 或 `.docx` 中解析 `错词:正词`、`错词=正词`、`错词 正词` 等多种格式。
     - 将解析结果放入“导入预览”列表。
   - “✅ 更新导入词库”：
     - 将预览列表（或仅选中项）合并到 `Word_Library.txt`。
     - 内置去重逻辑：不会重复写入已存在的 `(错词, 正词)` 组合。
   - “词库”Tab：
     - 直接从 `Word_Library.txt` 读取全部词条，并支持排序浏览。
   - “文档”Tab：
     - 打开 `Word_Library.md` 并展示为只读 Markdown 文本视图。
     - 在展示时动态替换 `创建时间` / `最后更新` 的日期字符串为当前时间（仅 UI 显示，不会回写文件），方便查看时识别“当前查看时间”。

## 知识库体系设计

### 1. 文件角色与关系

- **`Proper_Word_Library.txt`（专业术语库）**
  - 每行一个术语，如：`复合体灵`、`灵体` 等。
  - 作为高质量“正确词”的来源，用于：
    - 人工维护核心术语集合。
    - 基于字符映射 `CHAR_MAPPING` 自动生成错词候选。

- **`Word_Library.txt`（主词库）**
  - 格式：每行一个 `错误词=正确词`。
  - 通过以下途径累积内容：
    - 从素材文件（如 `素材文件/library.txt`）导入。
    - 在知识库管理器中手工导入。
    - 从“待审核候选错误词”中筛选通过后批量导入。
  - 是所有校对逻辑的“权威来源”，`Word_Library.md` 与 Word 校对工具均基于此内容。

- **`Word_Library.md`（词库文档视图）**
  - 从 `Word_Library.txt` 同步生成的 Markdown 文档，更适合人工阅读。
  - 在 `## 词条列表` 下，将词条按 A-C/D-J/K-Z 等分组显示。
  - 附带示例与“如何添加新词条 / 如何自动同步”的说明。
  - `word_reader.py` 在运行时也从此文件中解析 `错误词=正确词` 对。

- **`Pending_Review.txt`（待审核候选错误词）**
  - 预留文件，用于在未来将 GUI 中的“待审核”内容落盘。
  - 当前项目版本未主动写入，仅作格式声明：`错误词=正确词`。

- **`素材文件/library.txt`**
  - 原始词库素材示例，包含大量匹配对。
  - 实际用法：
    - 可作为批量导入的数据源。
    - 有助于初始化或扩充 `Word_Library.txt`。

### 2. 更新与同步流程

以下是一个推荐的知识库维护工作流：

1. **维护专业术语**
   - 在知识库管理器的“术语”Tab 中，通过“➕ 添加专业术语”维护 `Proper_Word_Library.txt`。
   - 术语越完善，基于 `CHAR_MAPPING` 自动生成的错词候选就越丰富、越贴近实际语音/口音误差。

2. **从术语生成候选错词**
   - 在知识库管理器中：
     - 使用“🎯 全部生成候选错词”对所有术语生成候选，或
     - 使用“🎯 选中生成错词”仅针对部分术语生成。
   - 生成结果展示在“待审核”列表中，以 `错误词 → 正确术语` 形式显示。

3. **审核候选错词并导入主词库**
   - 在“待审核”Tab 中：
     - 点击每一条切换“☐ / ☑”，选择要导入的候选。
     - 使用“全选 / 取消全选 / 删除选中”等操作快速筛选。
   - 通过“📥 批量导入词库”将勾选的候选写入 `Word_Library.txt`（自动去重）。

4. **从外部文件手工导入**
   - 使用“📥 手工导入词库”，从 `.txt` 或 `.docx` 中解析词条到“导入预览”。
   - 确认无误后，使用“✅ 更新导入词库”写入 `Word_Library.txt`。

5. **同步生成 Markdown 词库文档**
   - 在命令行中运行（或双击）`src/errorLibrary/update_library.py`：
     - 从 `Word_Library.txt` 读取所有词条。
     - 识别出相对于 `Word_Library.md` 的新增部分。
     - 在 `Word_Library.md` 末尾追加“### 新增词条”段落，以 `错误词=正确词` 形式写入。

6. **在 Word 校对工具中使用**
   - `word_reader.py` 会在执行校准时：
     - 读取 `Word_Library.md`，解析其中所有 `错误词=正确词`。
     - 逐段落、逐单元格替换内容，并记录替换日志。
     - 生成新的 `xxx_校准.docx` 文件，并在日志中展示详细替换列表。

## 开发者/维护者建议

- **不直接手改 `Word_Library.md`**  
  优先通过 `Word_Library.txt` + `update_library.py` 和知识库管理器 GUI 维护词条，避免 Markdown 文本与源数据不一致。

- **新增术语前先确认规范**  
  `Proper_Word_Library.txt` 中的术语是自动生成错词候选的基础，建议保证：
  - 含义明确、无歧义。
  - 命名尽量稳定，避免频繁重命名。

- **调整错词映射规则时注意 `CHAR_MAPPING`**  
  错词生成逻辑集中在 `readLibrary.py` 的 `CHAR_MAPPING` 中，任何添加/修改都可能显著增加候选数量，应适当评估：
  - 是否会产生过多噪音匹配。
  - 是否符合实际用户的发音/输入习惯。

- **备份与版本管理**  
  项目当前不是 Git 仓库，建议在实际使用中：
  - 将整个项目目录纳入 Git 管理，尤其是 `Word_Library.txt` 与 `Proper_Word_Library.txt`。
  - 在重要更新（大批量导入词库或修改映射规则）前做一次提交或复制备份。
